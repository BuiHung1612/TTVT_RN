Quy ước và Linting

Quy ước đặt tên

  • PascalCase cho tên file và component:
    // Tốt
    const UserProfileScreen = () => { ... };
    // File: UserProfileScreen.tsx
  • camelCase cho file tiện ích, hooks, và service:
    // Tốt
    const useAuth = () => { ... };
    // File: useAuth.ts

Công cụ Linters & Formatters

ESLint

Phân tích mã tĩnh để xác định vấn đề về chất lượng mã, lỗi cú pháp và vi phạm quy tắc mã hóa.

Prettier

Định dạng mã tự động, duy trì phong cách mã hóa nhất quán, loại bỏ các tranh cãi không cần thiết về định dạng.

Ví dụ cấu hình .eslintrc.js

// .eslintrc.js
module.exports = {
  extends: ['react-native'],
  rules: {
    'react-native/no-inline-styles': true,
  },
};
